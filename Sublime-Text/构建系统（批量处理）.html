构建系统让你通过像外部程序运行你的文件 make，整理，翻译，等等。

可执行文件从生成系统称为必须在你的PATH。有关确保更多的信息，PATH看到崇高文本设置正确，请参阅故障排除构建系统。

文件格式

打造系统是JSON文件，并具有扩展_.sublime建造_。

例

下面是一个构建系统的一个例子：

{
    "cmd": ["python", "-u", "$file"],
    "file_regex": "^[ ]*File \"(...*?)\", line ([0-9]*)",
    "selector": "source.python"
}

CMD

必需的。这个选项包含要执行的实际命令行：

python -u /path/to/current/file.ext

file_regex
一个Perl样式的正则表达式来捕获从外部程序的输出错误信息。此信息用于帮助您浏览的错误实例与F4。
选择selector
如果工具|构建系统|自动，Tools | Build System | Automatic
选项设置，崇高的文本会自动匹配找到相应的编译系统的活动文件选择该文件的适用范围。

除了 ​​选项，您可以使用的编译系统的一些变数太多，因为我们上面做$文件，它扩展到活动缓冲区的文件名 ​​。

在哪里存储系统构建

构建系统必须设下的地方 Packages folder（Packages/User）。许多软件包包括自己构建系统。

运行系统构建

构建系统可以通过按运行F7或工具|构建，Tools | Build
