指令有一个特殊字符，如调用：，之后的所有文本将被指令进行解释。例如：

这指示崇高文本首先搜索相匹配的文件岛 ，然后去排队123。

侧边栏

侧边栏让你项目的概述。添加到侧边栏的文件和文 ​​件夹将在转到任何和项目范围内的行动。项目和侧边栏是密切相关的。总有一个开放的项目，无论是隐含或明确。

要打开或关闭侧边栏，按下Ctrl + K， Ctrl + B键。

侧边栏可以使用箭头键进行导航，但首先你需要给它输入焦点，按Ctrl键+ 0。要返回输入焦点缓冲区，按Esc键。或者，您可以用鼠标同样的效果，但为什么你会吗？

侧边栏还可以通过上下文菜单中提供了基本的文件管理操作。

工程

项目组组文件和目录，你需要努力为单位。一旦你建立你的项目适合您通过添加文件夹的方式，保存它，并给它起一个名字。项目文件使用_.sublime项目_扩展。

您可以添加和删除文件夹与工程项目的菜单和侧边栏的上下文菜单。Futhermore，您可以拖动文件夹到一个窗口，他们将被自动添加。

要保存项目，选择项目|项目另存为...。

为了项目，按之间快速切换的Ctrl + Alt + P。Alernatively您可以浏览项目|最近的项目。

您可以打开从一个项目的命令行通过传递 _.sublime项目_文件作为参数。

项目定义

项目定义存储在JSON文件与_.sublime项目_ 扩展。只要有一个_.sublime项目_文件，你会发现一个辅助_.sublime，工作区_文件也包含用户特定的数据，如打开的文件和修改每个。后者则用于通过崇高的文本，你不应该自己进行编辑。

项目定义支持三个顶层部分：文件夹folders，所包含的文件夹，设置settings，用于设置覆盖，并build_systems，对于具体项目的构建系统。举个例子：

{ 
    "folders" : 
    [ 
        { 
            "path" :  "src" , 
            "folder_exclude_patterns" :  [ "backup" ] 
        }, 
        { 
            "path" :  "docs" , 
            "name" :  "Documentation" , 
            "file_exclude_patterns" :  [ "*.css" ] 
        } 
    ], 
    "settings" : 
    { 
        "tab_size" :  8 
    }, 
    "build_systems" : 
    [ 
        { 
            "name" :  "List" , 
            "cmd" :  [ "ls" ] 
        } 
    ] 
}

文件夹
每个文件夹必须有一个路径path，并且可任选地具有一个 folder_exclude_patterns和file_exclude_patterns设置。路径可能是相对于项目目录或绝对路径。文件夹也可以给予名称设置，设置他们是如何边栏上显示。
设置

一个项目可以定义项目特定的设置，这仅适用于（开）项目中的文件。具体项目的设置将覆盖普通用户设置，但没有语法特定的设置。

您可以覆盖几乎所有的设置（不包括全局设置）。
