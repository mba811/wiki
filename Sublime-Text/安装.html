安装

安装卓异文本的过程是为每个平台不同。

请务必阅读使用条件的官方网站。崇高的文字是不是免费的。

32位或64位？

如果您正在运行的是64位操作系统，否则32位版本的64位版本。

Windows，如有疑问，请选择32位版本。现代的64位版本的Windows可以运行32位软件。

对Linux在终端运行这个命令来检查您的操作系统的类型：

uname -m

对于OS X，你可以忽略此节：只有一个版本的崇高文字的OS X.

Windows

便携式或无法移植？

崇高的文本提供了两种不同的Windows：正常，便于携带。如果您需要便携式安装，你可能已经知道。否则，请与正常之一。

正常的安装两个文件夹之间单独的数据：安装文件夹正确，以及_数据目录_。这些概念在本指南后面解释。正常的安装还集成了崇高的文本与Windows右键菜单。

便携式装置将保持崇高的文本需要在一个单一的文件夹中运行的所有文件。然后，您可以将这个文件夹四周，编辑器仍然可以工作。

如何安装Sublime Text的普通版

下载安装程序，双击它，然后按照屏幕上的说明。

如何安装Sublime Text的便携版

下载包并解压到您选择的文件夹。你会发现_sublime_text.exe_该文件夹内的可执行文件。

OS X

下载并打开_的.dmg_文件，然后拖动文本崇高2捆到_应用程序_文件夹中。

Linux的

您可以下载包并手动解压。另外，您也可以使用命令行。

对于I386

cd ~
wget http://c758482.r82.cf2.rackcdn.com/Sublime\ Text\ 2.0.1.tar.bz2
tar vxjf Sublime\ Text\ 2.0.1.tar.bz2

对于x64

cd ~
wget http://c758482.r82.cf2.rackcdn.com/Sublime Text 2.0.1 x64.tar.bz2
tar vxjf Sublime\ Text\ 2.0.1\ x64.tar.bz2

现在，我们应该将未压缩的文件到合适的位置。

sudo mv Sublime\ Text\ 2 /opt/

最后，我们创建了一个符号链接在命令行中使用。

sudo ln -s /opt/Sublime\ Text\ 2/sublime_text /usr/bin/sublime

在Ubuntu的，如果你也想崇高的文本添加到团结luncher，请继续阅读。

首先，我们需要创建一个新的文件。

sudo sublime /usr/share/applications/sublime.desktop

复制以下进去。

[Desktop Entry]
Version=2.0.1
Name=Sublime Text 2
# Only KDE 4 seems to use GenericName, so we reuse the KDE strings.
# From Ubuntu's language-pack-kde-XX-base packages, version 9.04-20090413.
GenericName=Text Editor

Exec=sublime
Terminal=false
Icon=/opt/Sublime Text 2/Icon/48x48/sublime_text.png
Type=Application
Categories=TextEditor;IDE;Development
X-Ayatana-Desktop-Shortcuts=NewWindow

[NewWindow Shortcut Group]
Name=New Window
Exec=sublime -n
TargetEnvironment=Unity

如果您已经注册了崇高的文本复印件，但每次你打开它时你输入的许可证要求，你应该尝试运行此命令。

sudo chown -R username:username /home/username/.config /sublime-text-2

只需更换用户名与您帐户的用户名。这应该可以解决的，你打开了崇高的文本以root当您第一次输入的许可证的情况下，允许误差。

危险之中......或者不

sublime-text有三种版本：

  • 稳定Stable（默认）
  • 开发Dev
  • 每夜Nightly

此外，还有为sublime-text3测试版是只提供给谁拥有许可证的用户单独通道：

  • 3-β（相当于_每晚_）
  • 3-DEV

如果你正在一个NASA项目或在紧张的最后期限，继续使用稳定版本，并停止阅读这里的稳定版本是更好的测试和更可靠的比别人日常使用。大多数用户将要使用的稳定版本而已。

在_开发_和_夜间_的渠道是不稳定的，这可能意味着构建通过他们发布的包含错误，并不能可靠地工作。他们更新往往比稳定版本。

开发版本可供大家和发布插图中的稳定版本。虽然没有完全准备好但日常使用，他们展示的新功能的大多是不间断的方式。

最后，每晚构建的最前沿，与频繁的更新，也不同严重程度的频繁问题。他们是有趣的尝试，但这样做需要您自担风险。Nighlty版本是只对注册用户。
